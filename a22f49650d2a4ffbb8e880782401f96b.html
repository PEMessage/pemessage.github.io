三.Verilog

FPGA Note

一.HDL简介

  1. Hardware Description Language HDL 硬件描述语言
    1. 定义: 可以从不同层次描述数字电路结构和功能 不同层次: 指高层次(类C) 底层(类汇编)
    2. 和C异同
      1. 并行性: 各个模块同时开始工作,而C语言是串行
      2. 属于高级硬件描述,语法上类C
    3. 数字系统设计: 逻辑设计(前端)→电路实现(后端)→验证
      1. 前端: HDL编程,逻辑设计部分
      2. 后端: HDL→综合工具→门级别网表→和某种工艺的基本元件再对应→布局布线工具→电路结构
    4. 常见HDL
      1. 美中: Verilog HDL
      2. 欧洲: VHDL
      3. FPGA设计也可采用原理图输入法

二.Verilog基础

  1. 逻辑值:
    1. 0: 表示GND
    2. 1: 表示VCC
    3. X: 表示未知电平,但是1 或 0中一个
    4. Z: 表示高阻态,外部无激励,属于悬空,不能判断1或0或其中任何中间
    5. 非1的其他值均为逻辑假
  2. 数字格式
    1. [位宽] ' [d(dec)\b(binary)\h(16进制)\oOct)] 值
    2. 值中间可用下划线分割如 16'b1001_1010_1111_1001
    3. 如果没有指定 位宽和进制 默认为32'd
  3. 标识符
    1. 可用于定义 模块名 端口名 信号名(可理解为变量或常量)
    2. 可为 [a-z A-Z 0-9 $ _] 第一个字符为 字母或下划线
    3. 区分大小写:
    4. 代码规范
      1. 不建议大小写混合 内部信号全小写
      2. 下划线区分: 如cpu_addr
      3. 采用前缀或后缀: 如clk_50m clk_cpu
  4. 数据类型
    1. 类型简介
      1. 寄存器 线网 参数,其中寄存器 线网有实际的电路映射
    2. 寄存器
      1. 抽象数据存储单元,可以被幅值
      2. 初始值为 X(参见上面逻辑值)
      3. 赋值
        1. 只能被initial always赋值
        2. always既可以组合逻辑也可时序逻辑
        3. 若语句为时序逻辑(即含有时钟信号),则实际上对应一个触发器
        4. 若仅为组合逻辑,对应实际连线
    3. 线网
      1. 表示结构实体(门)之间的物理连线
      2. 默认值为 Z
      3. 不指定数据类型默认为wire类型
      4. 不能存储值,值由驱动线网的器件决定
      5. 驱动线网的元器件
        1. 门
        2. 连续赋值语句
        3. assign语句: 仅能描述组合逻辑
    4. 参数 parament
      1. 实际为常量
      2. 可用于表示状态机状态 数据位宽 延迟大小
      3. 模块调用时候可以传递参数
  5. 运算符
    1. 算数运算符: /小数部分省略
    2. 关系运算符:
    3. 位移运算符: 左右移动都是填0来补空缺
    4. 位拼接运算符: c = { a, b[3:0] }

三.Verilog语法

  1. 注释: 类C // 或 /*...*/
  2. 程序框架
    1. 基本设计单元为Block 模块:
      1. 一部分描述接口 一部分描述逻辑功能
      2. 程序包括: 端口定义 IO说明(可以和端口定义合并) 内部信号声明 功能定义
      3. 不可综合模块: 不对应实际的电路,例如仿真文件
    2. 接口描述
      🖥️接口描述示例
      module block(a,b,c,d);
	input a, b;
	output c, d;

assign c = a | b ; /未指明类型默认为wire
assign d = a & b ;

endmodule
    3. 功能定义
      1. assign: 描述组合逻辑
      2. always: 描述组合时序
      3. 实例化器件: 如 and #2 u1(q,a,b)
      4. 三种逻辑功能并行
  3. 参数传递
    🖼️参数传递示例
    1. 除了从端口中将信号连接起来之外,也可以传递参数(parameter)
  4. always/initial语句
    1. 简介
      1. 其中包含的为过程块,在过程块中可对变量赋值
      2. 不允许在多个always中对同一个变量赋值: 多个always并行性
      3. 不要混合使用阻塞赋值非阻塞赋值
    2. 条件触发
      1. always @(posedge sysclk or negedge sys_rst) begin ... end
      2. @(..) 成为敏感列表
    3. 时序逻辑
      1. 边缘触发往往为时序逻辑
      2. 非阻塞赋值: 计算RHS在先,只有在过程快结束时才更新LHS—一般为时序逻辑
    4. 组合逻辑
      1. 电平触发往往为组合逻辑
      2. 在组合逻辑中@( * ) 表示对过程块中所有变量的变化都为敏感的
      3. 阻塞赋值: 计算RHS(左值)并更新LHS(右值)—一般为组合逻辑
    5. 过程块语句
      1. if ... else ...
        1. 只能在过程块中使用,如果需要包含多个语句 需要begin end(类似c语言的 { } )
        2. if else 配对使用,只有if可能产生latch(锁存器)
          1. latch只在组合逻辑中出现,会造成毛刺较多(???)
      2. case( num ) ... endcase:
        🖼️case语句示例
        1. casez : 比较时候,不考虑分支表达式中z,其他位置相等即可
        2. casex : 不仅z不比较 x值也不比较
        3. 分支表达式 和 控制表达式的位宽必须完全相等
        4. casse的分支表达式没有给出全部可能时候,也会造成latch

四.状态机

  1. 简介
    🖼️状态机模型
    1. 状态机模型中分为两个组合逻辑 一个时序逻辑
    2. Verilog为并行执行的,不好做顺序逻辑

  2. 状态机设计:

    3. 状态跳转(时序逻辑)
      🖼️状态跳转示例
    4. 下个状态判断(组合逻辑)

      🖼️状态输出的两种方法

  3. 状态机扩展
    🖼️扩展状态机图示
