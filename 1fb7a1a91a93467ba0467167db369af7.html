  2. Vivado硬件调试
    1. 硬件调试简介
      1. 通过向FPGA中添加ILA VIO核实现硬件调试,并通过Jtag连接到PC
      2. Integrate Logic Analyzer(ILA): 监控内部信号和端口信号
      3. Virtual Input/Output(VIO): 实时监控驱动内部信号和端口信号
    2. ILA 内部逻辑分析器
      🖼️ILA添加方法
      1. 添加方法
        1. HDL方法
        2. 综合之后,在网表(Netlist)中添加
    3. HDL方法
      1. IP Integrated
        1. 搜索ILA: System ILA不使用,用于IP"集成器"中,我们使用的的是IP的"调试设计"
        2. 选择模式: AXI Native(default)
        3. 选择probe数目和存储深度
        4. 配置Probe_Port位宽
        5. 选择OK
        6. 后配置Synthesis综合选项: Global和Out of context per IP(OOC模式)
        7. 选择OOC 时候,创建时候会被综合一次,综合整个工程时候不会被综合,Global模式每次都会被综合
      2. 例化
        🖼️例化图示
        1. 代码中"例化"将代码中端口同IP核中端口想连接
        2. Source中选择IP Source
        3. 选择instantiation Template中veo文件
        4. 将例化模板拷贝到Designed文件
        5. 修改模板中的例化名 一般u_模块名
        6. 修改probe0 probe1 probe2 clk
    4. 综合方法
      🖼️添加属性
      1. Synthesis之后,在网表里对应信号右键添加Mark Debug
      2. 在Designed源文件中,添加属性,防止软件优化部分信号,并且自动将信号设为Debug
      3. 💡单纯在Designed中添加属性,而不在综合后网表里Setup Debug,不影响比特流
      4. 综合之后,相当于还是在网表文件中做修改(.xdc)
      5. Setup Debug 选择观察的信号,并右键分配时钟(类似ILA设置)
      6. 设置采样设置等等

  3. Vivado Simulator使用
    1. 仿真概述
      1. 硬件调试: 需要消耗FPGA资源
      2. 软件仿真: 一般不需要,且仿真需要提供Test Bench(tb)激励文件
      3. 参考文件: Vivado设计套件使用指南-仿真篇(UG900)
      4. 仿真分类: RTL仿真(仅针对代码功能,不考虑门延迟 线延迟) 时序仿真 常规只需要功能仿真
      5. 此处仿真使用(第一方Vivado Simulator)
    2. 创建TestBench激励文件
      🖼️创建激励文件图示
      1. 在Source-Simulator Source里右键Add Source
      2. Create File: 一般文件命名tb_模块名.v
      3. 对于激励文件里自带的module,不需要输入输出端口
      4. 声明reg变量 wire变量
      5. 初始化initial begin...end
        1. 1'代表位宽 b代表进制
        2. #200 指200ns由前面的timescale确定
        3. timescale [单位]/[精度]
      6. 代码编写
        1. always #10 sysclk=~sysclk
      7. 代码例化
        🖼️例化图示
        1. 相当于调用模块,并在对应接口处连线
    3. 开始仿真
      1. 仿真选项
        1. 行为仿真 综合功能仿真 综合时序 实现功能仿真 实现时序仿真
        2. 功能仿真实际上为一般指→行为方法
        3. 时序仿真实际上一般指→实现时序仿真
        4. Setting中可修改默认的仿真时间
        5. 修改代码后(Designed)直接Relaunch Simulation即可
      2. Scope:
        1. tb模块(文件)
        2. u例化模块: 此时以激励文件位主体,Designed文件是作为module被tb调用的
        3. glbl 全局文件
      3. Object: 默认只显示tb文件信号,需要u模块信号,需要在Scope中选中模块,并右键添加
